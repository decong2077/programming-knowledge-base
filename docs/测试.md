# 测试

## 概述
测试是保证代码质量和防止回归的重要手段。

## 测试金字塔


## 单元测试
测试单个函数或方法的行为。

### 测试框架
- **Python**: pytest, unittest (参见 [Python](Python.md))
- **JavaScript**: Jest, Vitest (参见 [JavaScript](JavaScript.md))
- **Go**: 内置 testing 包 (参见 [Go](Go.md))

### 测试示例结构


## 集成测试
测试多个组件协作是否正确。
- API 接口测试
- 数据库交互测试
- 外部服务调用测试

## E2E 测试
端到端测试,模拟用户真实操作。
- Cypress, Playwright
- 覆盖核心用户流程

## 测试最佳实践
1. **测试行为,不测试实现**
2. **每个测试只测一件事**
3. **测试命名清晰** (should_xxx_when_yyy)
4. **保持测试独立**
5. **使用 Mock 隔离依赖**

## 测试覆盖率
- 目标: 70-80% (非 100%)
- 关注关键路径覆盖
- 覆盖率不是质量唯一指标

## TDD (测试驱动开发)
1. 先写失败的测试
2. 写最少代码让测试通过
3. 重构代码
4. 重复以上步骤
