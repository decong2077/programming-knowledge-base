# 代码规范

## 概述
代码规范是团队保持一致代码风格的约定,提升可读性和可维护性。

## 为什么需要代码规范
- **可读性** - 降低理解成本
- **协作效率** - 减少风格争论
- **代码审查** - 聚焦逻辑而非格式
- **新人上手** - 统一的代码风格

## 各语言规范工具

### Python
- **PEP 8** - 官方风格指南
- **Black** - 自动格式化
- **Flake8** - 风格检查
- **isort** - 导入排序

### JavaScript / TypeScript
- **ESLint** - 代码检查
- **Prettier** - 代码格式化
- **StandardJS** - 零配置规范

### Go
- **gofmt** - 官方格式化工具
- **golangci-lint** - 综合检查

## 核心规范要点

### 命名规范
- 变量: 有意义、动词+名词
- 函数: 动词开头 (get_user, is_valid)
- 常量: 全大写 (MAX_SIZE)
- 类: 大驼峰 (UserService)

### 代码组织
- 函数长度: 不超过 50 行为佳
- 参数数量: 不超过 3 个
- 文件长度: 控制在 300 行内
- 模块职责单一

### 注释规范
- 解释为什么,而不是做什么
- 公共 API 必须注释
- 复杂逻辑添加注释
- 避免无用注释

## 自动化检查
- 集成到 [版本控制](版本控制.md) pre-commit
- CI 流程中自动检查
- IDE 实时提示

## 重构原则
- 童子军规则: 离开时比来时更干净
- 小步重构,频繁提交
- [测试](测试.md) 保障重构安全
