# 版本控制

## 概述
版本控制系统记录文件变更历史,支持多人协作开发。

## Git 基础
### 核心概念
- **仓库 (Repository)** - 存储文件的目录
- **提交 (Commit)** - 一次变更记录
- **分支 (Branch)** - 独立的开发线
- **合并 (Merge)** - 整合分支

## Git 工作流

### Feature Branch 工作流
1. 从 main 创建 feature 分支
2. 在分支上开发
3. 提交并推送到远程
4. 创建 Pull Request
5. 代码审查后合并到 main

### Git Flow
- main: 生产环境代码
- develop: 开发环境
- feature/*: 功能分支
- release/*: 发布分支
- hotfix/*: 热修复分支

## 提交规范
### Conventional Commits


类型:
- feat: 新功能
- fix: 修复 bug
- docs: 文档更新
- style: 代码格式
- refactor: 重构
- test: 测试相关
- chore: 构建/工具相关

## 协作最佳实践
1. **频繁提交** - 小步提交,易于审查
2. **清晰描述** - 提交信息说明变更内容和原因
3. **及时同步** - 经常 pull 避免冲突
4. **代码审查** - PR/MR 机制
5. **保护主分支** - 禁止直接推送
